# GitHub Desktop - Quick Guide

## Step 1: Verify Repository Connection

1. Open **GitHub Desktop**
2. Check that your repository is listed in the left sidebar
3. You should see "FYP" or your repository name

## Step 2: Stage All Files

1. In GitHub Desktop, you should see all your files listed in the left panel
2. Look at the bottom left - you should see files ready to commit
3. If files aren't showing:
   - Click the **"Changes"** tab at the top
   - All modified/new files should appear there

## Step 3: Commit Files

1. At the bottom left, you'll see a text box for the commit message
2. Enter a commit message:
   ```
   Initial commit: Facial recognition login system with BCE framework
   ```
3. Click the **"Commit to main"** button (or "Commit to master" depending on your branch name)

## Step 4: Push to GitHub

1. After committing, click the **"Push origin"** button at the top
2. Or go to **Repository** → **Push**
3. Wait for the upload to complete
4. You should see a success message

## Step 5: Verify on GitHub

1. Click **"View on GitHub"** button (or go to your repository on github.com)
2. Verify all files are uploaded correctly
3. Check that README.md displays properly

## Troubleshooting

**If you don't see files to commit:**
- Make sure you're in the correct repository folder
- Try refreshing: **Repository** → **Refresh**

**If push fails:**
- Check your internet connection
- Make sure you're logged into GitHub Desktop
- Try: **Repository** → **Push** again

**If you see merge conflicts:**
- Usually won't happen on first commit
- If it does, contact me for help
